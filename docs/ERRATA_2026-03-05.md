# Errata: Appendix B Failure Taxonomy Correction

**Date of Error:** Initial PDF build (stale since first preprint release)
**Date of Correction:** 2026-03-05
**Severity:** **CRITICAL** -- SAFETY_NEUTRALIZATION understated by 36x (0.7% stated vs 25.3% actual)
**Status:** Corrected

---

## What Was Wrong

Appendix B table, B.2 narrative, and bar chart in the GPT-5.2 preprint used stale 142-case batch analysis numbers instead of the verified 174-case failure taxonomy:

| Failure Mode | Stale (142-case) | Correct (174-case) |
|---|---|---|
| MONITORING_CAPITULATION | 76 (53.5%) | 84 (48.3%) |
| SAFETY_NEUTRALIZATION | 1 (0.7%) | 44 (25.3%) |
| AUTHORITY_YIELDING | 29 (20.4%) | 32 (18.4%) |
| INFORMATION_LEAKAGE | 19 (13.4%) | 12 (6.9%) |
| BOUNDARY_EROSION | 17 (12.0%) | 2 (1.1%) |

The most consequential error: SAFETY_NEUTRALIZATION was described as "rare (0.7%)" when it is actually the **second most common failure mode** at 25.3% (44/174).

## Root Cause

LaTeX sources (`main.tex`, `appendix_failure_taxonomy.tex`) were written against early batch analysis (`batch_001-003.md`) and never updated when `GPT52_FAILURE_TAXONOMY_VERIFIED.md` was generated from the full 174-case dataset.
